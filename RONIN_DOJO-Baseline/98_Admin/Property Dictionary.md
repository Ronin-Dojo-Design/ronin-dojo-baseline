# Property Dictionary

Use these properties consistently.

| Property | Meaning |
|---|---|
| `note_type` | The kind of note: curriculum_unit, technique, lesson_plan, class_note, mood_note, student_note |
| `status` | Workflow state: draft, review, ready_to_publish, published |
| `review_status` | Optional finer state such as draft, internal_review, approved |
| `publish_target` | Where the note may go later, such as `pods:curriculum` or `pods:technique_library` |
| `source_of_truth` | Usually `obsidian_planning`; use this to remind yourself what is planning vs canonical |
| `visibility_scope` | private_self, instructor_only, shared_teaching_team, org_staff, student_visible |
| `module` | Your course or curriculum module number |
| `discipline` | Striking, grappling, weapons, forms, etc. |
| `style` | Style or subsystem |
| `energy_level` | Reflection score for mood notes |
