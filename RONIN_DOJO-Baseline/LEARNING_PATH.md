# Obsidian Learning Path for Ronin

## Module 1 - Learn the floor
**Outcome:** You understand vaults, folders, links, tags, and Properties.

Practice:
- Create one note in `03_Curriculum`.
- Add links to one technique and one lesson plan.
- Add properties for `status`, `note_type`, and `publish_target`.

## Module 2 - Build repeatable notes
**Outcome:** You stop starting from blank pages.

Practice:
- Use the daily note template.
- Use the curriculum unit template.
- Use the class note template right after a session.

## Module 3 - Put tasks where the truth lives
**Outcome:** Your tasks live inside source notes and are collected by dashboards.

Practice:
- Add three tasks to a curriculum note.
- Add two tasks to a lesson plan.
- Review the task dashboard in `02_Dashboards/Command Center.md`.

## Module 4 - Learn the query layer
**Outcome:** You can see your vault as structured data.

Practice:
- Use Dataview table queries in the dashboard.
- Edit a property and watch the table update.
- Open the optional Bases files and compare the feel.

## Module 5 - Start capture automation
**Outcome:** New notes start in the right shape with less manual work.

Practice:
- Enable Templater folder templates.
- Create one class note and one mood note from templates.
- Use one Obsidian URI from `98_Admin/Obsidian URI Examples.md`.

## Module 6 - Connect to the wider Ronin system
**Outcome:** You know what should stay in Obsidian and what should be published elsewhere.

Practice:
- Mark one technique `ready_to_publish`.
- Mark one curriculum unit `ready_to_publish`.
- Use `publish_target` and `visibility_scope` consistently.

## Module 7 - Weekly review rhythm
**Outcome:** You maintain the system instead of letting it drift.

Practice:
- Open `02_Dashboards/Weekly Review.md`.
- Clear inbox notes.
- Review any note still in `draft` or `review`.
- Decide what advances, what archives, and what becomes a real task.
