# Ronin Dojo Baseline Systems Pack

This pack updates the old monorepo control-docs for the **new** repo:
`Ronin-Dojo-Design/ronin-dojo-baseline`

## Included
- repo audit
- truth index
- alias ledger
- manual-boundary registry
- JETTY 3.0 systems profile
- usage guide
- next-session loading order
- SOPs for data flows and wiring flows
- E2E user lifecycle with ASCII diagrams
- agent workflows, rituals, and protocol folder map
- content engine command center and intake system
- video shortcut + Iggy automation flow

## Why this pack exists
The old docs assumed:
- WordPress
- Pods
- brand-specific WP themes/plugins
- legacy RoninDashboard control plane

The new repo is different:
- Next.js web app
- Expo mobile app
- Postgres + Prisma
- Better-Auth
- multi-brand by `brand` column
- repo-native wiki, sessions, rituals, and protocols

So the governance, truth map, content system, and rituals had to be re-centered around this repo.
