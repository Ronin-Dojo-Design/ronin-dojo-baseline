## Petey Plan — White-Label + Brand Ops Lane

### Problem statement

The platform is functionally complete at 80-90% across school ops, tournaments, content, and curriculum. But the entire UI — header, footer, hero, site name, navigation, color palette — is the Dirstarter template default. There are zero visual cues that this is a martial arts platform. No nav links to Programs, Tournaments, Courses, Directory, or Techniques. The homepage shows a "Tools" directory with a CTA for "Submit your tool."

This is the **40% white-label gap** identified in SESSION_0060 and the critical path to May 18 four-brand launch.

### Scope

5 work packages, ordered by dependency and impact:

---

#### WP-1: Brand-aware site config + i18n (foundation)

**What:** Make `siteConfig`, `brand.json`, and `navigation.json` brand-aware so all downstream components render correct names, taglines, and nav labels.

**Files to create/modify:**
- `config/site.ts` → export a `getBrandSiteConfig(brand)` function instead of a static object
- `messages/en/brand.json` → replace Dirstarter copy with Baseline Martial Arts copy (first brand)
- `messages/en/navigation.json` → add keys: `programs`, `tournaments`, `courses`, `techniques`, `directory`, `schedules`

**Done criteria:** `siteConfig.name` returns `"Baseline Martial Arts"` when brand is `BASELINE_MARTIAL_ARTS`. Navigation i18n keys exist for all martial arts sections.

---

#### WP-2: Header + footer navigation (user-facing)

**What:** Replace the Dirstarter "Browse → Tools/Categories/Tags" nav with martial arts sections. Keep L1 patterns (NavLink, DropdownMenu, i18n keys).

**Header nav structure:**
```
Programs | Tournaments | Techniques | Directory | About
```
With a "Browse" dropdown containing:
```
Courses | Techniques | Directory
```

**Footer nav:** Mirror header sections + add Blog, Contact.

**Files to modify:**
- `components/web/header.tsx` — replace nav links
- `components/web/footer.tsx` — replace nav links
- Mobile nav (bottom of header.tsx) — same links

**Done criteria:** All martial arts pages reachable from the header/footer. No Dirstarter "Tools" references remain.

---

#### WP-3: Homepage + hero (brand landing)

**What:** Replace the Dirstarter "Launch Your Directory" hero with a Baseline Martial Arts landing page. Show featured programs, upcoming tournaments, and a CTA to browse or sign up.

**Files to modify:**
- `app/(web)/(home)/hero.tsx` — branded tagline, description, CTA
- `app/(web)/(home)/page.tsx` — replace `ToolQuery` with featured programs/tournaments
- `app/(web)/(home)/count-badge.tsx` — show discipline/program/student counts instead of tool count
- `messages/en/brand.json` — Baseline tagline + description

**Done criteria:** Homepage shows Baseline Martial Arts branding with links to programs and tournaments. No "Tools" or "directory website" copy.

---

#### WP-4: Per-brand CSS theme tokens

**What:** Add brand-specific color palettes via CSS custom properties. The brand is resolved server-side (middleware `x-brand` header) and injected as a `data-brand` attribute on `<html>`.

**Approach (per ADR 0004 + L3 architecture):**

```css
/* styles.css — default (Baseline) */
:root { --color-primary: hsl(220 80% 50%); /* blue */ }

/* Brand overrides */
[data-brand="RONIN_DOJO_DESIGN"] { --color-primary: hsl(0 0% 15%); /* dark */ }
[data-brand="BBL"]               { --color-primary: hsl(45 90% 50%); /* gold */ }
[data-brand="WEKAF"]             { --color-primary: hsl(350 80% 45%); /* red */ }
```

**Files to modify:**
- `app/layout.tsx` — add `data-brand` attribute to `<html>` (read from `getRequestBrand()`)
- `app/styles.css` — add brand color overrides after default theme
- `components/web/ui/logo.tsx` — render brand-specific logo/wordmark

**Done criteria:** Each brand has a distinct primary color. Logo changes per brand. No hardcoded colors in components.

---

#### WP-5: Admin brand switcher (admin-only, ADR 0008)

**What:** Build the brand switcher UI for admins and multi-brand members. Sets `activeBrandId` in session. Deferred to last because the middleware already resolves brand from host — the switcher is an admin convenience.

**Files to create:**
- `components/admin/brand-switcher.tsx` — dropdown showing available brands
- Server action to update `session.user.activeBrandId`

**Done criteria:** Admin can switch brand in the admin panel. Admin queries scope to the selected brand.

### Risks

1. **L1 pattern drift:** Header/footer are L1 Dirstarter components. Modifying them is an extension, not a replacement. Keep the component structure, just change the links and labels.
2. **i18n key sprawl:** Keep martial arts nav keys in `navigation.json`, not scattered across feature-specific files.
3. **Theme flash:** `data-brand` must be set server-side in the root layout, not client-side, to avoid FOUC.

### Open decisions for operator — RESOLVED

1. **Logo assets:** Text-only wordmarks for now. Real SVG/PNG assets exist but deferred to polish pass.
2. **Homepage content strategy:** Static marketing copy first. Real seeded data (programs, tournaments) in a later session.
3. **"Tools" concept:** Keep as "Training Resources." Explore renaming/repositioning in SESSION_0062.
